Getting Started

Go from zero to pattern-aware oracle in five steps.

Install the toolkit

Add the package to your project with npm (or yarn / pnpm).

npm install remembrance-oracle-toolkit

Or, if you prefer Yarn:

yarn add remembrance-oracle-toolkit

Your first search

Import the oracle and search for patterns already harvested from your codebase. The search is fuzzy and context-aware. 

const { Oracle } = require('remembrance-oracle-toolkit');

const oracle = new Oracle();

// Search for patterns matching "rate limiter"
const results = await oracle.search('rate limiter');
console.log(results);
// [
//   { id: "P-017", name: "token-bucket-limiter", score: 0.94, ... },
//   { id: "P-031", name: "sliding-window-rate", score: 0.87, ... }
// ]

Score code coherency

The coherency scorer evaluates how well a piece of code aligns with your project's established patterns. Scores range from 0.0 (no alignment) to 1.0 (perfect). 

const code = `
  function retry(fn, attempts = 3) {
    for (let i = 0; i < attempts; i++) {
      try { return fn(); }
      catch (e) { if (i === attempts - 1) throw e; }
    }
  }
`;

const score = await oracle.computeCoherencyScore(code, {
  language: 'javascript'
});

console.log(score);
// { coherency: 0.82, patterns: ["retry-with-backoff"], suggestions: [...] }

Resolve a pattern

When you know what you need but not the exact implementation, ask the oracle to resolve it. It returns the highest-scoring match along with ready-to-use code. 

const pattern = await oracle.resolve({
  description: 'retry with backoff'
});

console.log(pattern.name);       // "exponential-backoff-retry"
console.log(pattern.code);       // Full implementation
console.log(pattern.coherency);  // 0.96

Docker setup

For the full ecosystem — including the Void Compressor, Reflector, and Swarm services — spin everything up with Docker Compose. 

# Clone the repo (if you haven't already)
git clone https://github.com/your-org/remembrance-oracle-toolkit.git
cd remembrance-oracle-toolkit

# Start all services
docker compose up -d

# Verify everything is running
docker compose ps

# Check the oracle health endpoint
curl http://localhost:3000/health

The docker-compose.yml at the repo root wires up the Oracle MCP server, Void Compressor, Reflector, and Swarm coordinator. See the Architecture page for how they connect.
