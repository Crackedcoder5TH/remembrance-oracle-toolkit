Features

🔍

Search & Resolve

Describe what you need — the oracle searches its memory and returns proven code with a PULL, EVOLVE, or GENERATE decision.

🔄

Self-Healing Code

Patterns are iteratively healed until they pass the Covenant — 15 principles of code quality, scored across 5 dimensions.

🌐

Three-Tier Storage

Local, Personal, and Community stores. Code flows from project to private library to shared kingdom — always with test proof.

🧪

Candidate Pipeline

Language variants and refinements are spawned automatically. Candidates are promoted only when they pass tests.

🤖

MCP Server

10 focused tools exposed via the Model Context Protocol — search, resolve, submit, register, sync, and more from any AI client.

How It Works

1

Search

Query the oracle before writing new code. It searches across all tiers for matching proven patterns.

2

Resolve

The oracle returns PULL (use as-is), EVOLVE (adapt a match), or GENERATE (write new).

3

Heal

Code is iteratively refined against the Covenant until it reaches optimal coherency across all dimensions.

4

Prove

Every pattern must have passing tests. No exceptions — the oracle only remembers what is proven.

5

Remember

Proven code is stored, synced, and shared. The kingdom's library grows with every contribution.
