Oracle Toolkit

Pattern Memory — Remembers code that worked and finds it when you need it

The core of the ecosystem. The Oracle stores only code that has passed tests, passed security checks, and scored above a quality threshold. When your AI needs a pattern, it pulls proven code instead of guessing. The library grows automatically with every commit.

PULL / EVOLVE / GENERATE smart retrieval decisions
302+ built-in patterns across 10 languages
7-dimension quality scoring (0–1.000)
15-principle Covenant safety filter
Three-tier storage: local, personal, community
Semantic and keyword search modes
Git hooks for automatic pattern capture
MCP server with 12 focused tools
const { RemembranceOracle } = require('remembrance-oracle-toolkit'); const oracle = new RemembranceOracle(); // Search for proven patterns const results = oracle.search('rate limiter', { mode: 'semantic', limit: 5 }); // Smart retrieval — pull, evolve, or generate const decision = oracle.resolve({ description: 'debounce function for React', language: 'javascript', }); // { decision: 'PULL', pattern: { name: 'debounce', coherency: 0.925 } } // Submit code — it must prove itself oracle.submit(code, { description: 'Merge sort implementation', tags: ['sort', 'algorithm'], testCode: testSuite, });

Reflector

Self-Healing CI — Scans PRs and auto-fixes weak code in seconds

Every pull request gets scored automatically. Code below the quality threshold is iteratively healed — the Reflector runs the SERF reflection loop up to 3 times, improving the code until it meets standards. No more "fix lint warnings" follow-up PRs.

Automatic PR scoring on every push
SERF reflection loop (up to 3 iterations)
Covenant safety filter blocks harmful code
Multi-repo snapshot and comparison
GitHub Action ready
Coherency gate: rejects code below 0.6 threshold

Agent Swarm

Multi-Agent Review — Multiple AI models review code and vote on the best version

Instead of relying on one AI model, Swarm orchestrates up to 7 different AI agents to review, score, and improve your code independently. They vote on the best version using consensus. The result is more robust than any single reviewer.

Up to 7 AI agents review simultaneously
Consensus voting on best version
Swarm code review via CLI or MCP
Swarm healing for complex refactors
Configurable providers and models
Detailed review breakdown per agent
# Multi-agent code review $ oracle swarm review --file src/api/handler.js # Heal code via swarm consensus $ oracle swarm heal --file src/utils.js # Check swarm status and providers $ oracle swarm status $ oracle swarm providers
